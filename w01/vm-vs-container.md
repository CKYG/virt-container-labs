# VM vs Container 比較

## 一、VM 與 Container 差異

| 比較項目 | VM（虛擬機） | Container（容器） |
|----------|--------------|------------------|
| 系統結構 | 每個 VM 都有自己的作業系統 | 共用同一個作業系統 |
| 啟動速度 | 比較慢 | 很快 |
| 資源使用 | 比較多 | 比較省 |
| 大小 | 通常很大（幾GB） | 比較小（幾MB） |
| 管理方式 | 用 VMware 這類軟體 | 用 Docker |


## 二、為什麼這堂課選擇「VM 裡跑 Docker」

我覺得用VM跑Docker比較安全，也比較穩定。因為VM像一台獨立的電腦，就算裡面出問題，也不會影響到我的電腦。
而且VM可以做snapshot，如果弄壞了可以直接還原。
Docker則是用來跑程式或服務，速度快而且輕量，所以兩個一起用剛好。

## 三、Hypervisor Type 1 vs Type 2

| 類型 | Type 1 | Type 2 |
|------|--------|--------|
| 安裝方式 | 直接裝在硬體上 | 裝在作業系統裡 |
| 效能 | 比較好 | 比較普通 |
| 使用地方 | 伺服器、雲端 | 一般電腦 |
| 範例 | ESXi | VMware Workstation |

這堂課用的是 VMware Workstation 是Type 2。  
因為我是在自己的電腦上做實驗用Type 2比較方便，不用特別的設備就能用。
雖然沒有Type 1那麼好，但對課堂練習來說已經夠用了。
